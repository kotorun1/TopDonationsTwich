# TOP DONATORS — Twitch Extension (CRT Terminal)

Твой оригинальный CRT-дизайн сохранён без изменений. Из статичной странички
получилось два экрана расширения плюс инструкция по бесплатному деплою.

## Что внутри

- **panel.html** — экран, который видят зрители в панели канала. Только
  читает данные и показывает **ТОП-10** донатеров, отсортированных по сумме.
  Обновляется в реальном времени, если стример меняет список.
- **config.html** — экран настройки, который видит **только стример**
  (вкладка "Configure" в его Twitch Extensions Dashboard). Тут можно
  добавлять, редактировать и удалять донатеров — всё сохраняется
  автоматически.
- Никакого своего сервера (EBS) не нужно — данные хранятся в **Twitch
  Configuration Service**, в сегменте `broadcaster`. Каждый стример,
  который установит расширение, получает свой собственный набор данных —
  они физически изолированы Twitch'ем per-channel.

```
Cloudflare Pages (хостит 2 статичных html-файла)
        │
        ▼
 Twitch Extension (panel.html + config.html)
        │
        ▼
 Twitch Configuration Service (broadcaster segment)
        │
        ▼
 Данные каждого стримера хранятся отдельно, бесплатно, без бэкенда
```

## 1. Деплой на Cloudflare Pages (бесплатно)

1. Создай новый репозиторий (GitHub/GitLab) и положи туда `panel.html` и
   `config.html` (можно в корень).
2. В Cloudflare Dashboard → **Workers & Pages** → **Create application** →
   **Pages** → **Connect to Git** → выбери репозиторий.
3. Build command оставь пустым, Build output directory — `/` (корень).
4. Deploy. После деплоя получишь домен вида
   `https://your-project.pages.dev`.
5. Открой `https://your-project.pages.dev/panel.html` и
   `https://your-project.pages.dev/config.html` в браузере — страницы
   должны открыться (в "STANDALONE MODE", это нормально вне Twitch).

**Важно:** Twitch требует, чтобы все ассеты расширения отдавались строго
по HTTPS — Cloudflare Pages это обеспечивает по умолчанию.

## 2. Регистрация расширения в Twitch

1. Зайди на https://dev.twitch.tv/console/extensions и нажми **Create
   Extension**.
2. Тип: **Panel** (панель под видео/в информации о канале).
3. Заполни название, описание, иконки (24/100/300 px — любые квадратные
   PNG под тему терминала).
4. Во вкладке **Capabilities**:
   - Panel Height: можно оставить дефолт (~300px под контент, наша
     таблица на 10 строк туда помещается; при желании увеличь высоту в
     настройках, страница адаптивна и растянется).
5. Во вкладке **Asset Hosting / Testing**:
   - **Viewer Path (panel.html)**: укажи
     `https://your-project.pages.dev/panel.html`
   - **Config Path (config.html)**: укажи
     `https://your-project.pages.dev/config.html`
   - Live Config Path можно указать тот же `config.html` — стример сможет
     редактировать список и во время стрима.
6. Сохрани, затем протестируй через **Extension Testing** (можно сразу
   активировать на свой тестовый канал) или через **Developer Rig**
   (https://github.com/twitchdev/developer-rig).

## 3. Как это работает без бэкенда

- `config.html` вызывает `Twitch.ext.configuration.set('broadcaster', '1',
  JSON.stringify({donators, updatedOn}))` — Twitch сам подписывает и
  сохраняет это на своей стороне для конкретного канала, когда страница
  открыта с ролью broadcaster (т.е. из Extensions Dashboard стримера).
- `panel.html` подписывается на `Twitch.ext.configuration.onChanged` и
  читает `Twitch.ext.configuration.broadcaster.content` — как только
  стример сохранил изменения, зритель видит обновлённый топ почти сразу.
- Так как хранилище — часть самого Twitch, тебе не нужен ни сервер, ни
  база данных, ни ключи API. Cloudflare Pages нужен только чтобы отдавать
  статичные `.html/.css/.js` файлы.

## 4. Функциональность

- ✅ CRT-дизайн — без изменений (тот же фон, шрифты, скан-линии, мигающий
  курсор, системные часы).
- ✅ Добавление донатеров (форма: имя, сумма, валюта).
- ✅ Редактирование (кнопка EDIT подставляет значения в форму).
- ✅ Удаление (кнопка DEL с подтверждением).
- ✅ Автосортировка по сумме — везде, всегда по убыванию.
- ✅ Автосохранение — каждое изменение сразу пишется в Configuration
  Service, отдельная кнопка "Save" не нужна.
- ✅ Просмотр ТОП-10 в зрительской панели (полный список виден только
  стримеру на экране конфигурации).
- ✅ Адаптивная вёрстка — работает и в узкой панели (320px), и в широкой.
- ✅ Полностью бесплатно: Cloudflare Pages (free tier) + встроенное
  хранилище Twitch, без стороннего хостинга/БД.

## 5. Ограничения, о которых стоит знать

- Twitch Configuration Service ограничивает содержимое одного сегмента
  примерно 5 КБ — этого хватает на несколько сотен донатеров, но если
  список станет огромным, стоит подумать про обрезку старых записей.
- `configuration.set` можно вызывать только когда `config.html` открыт с
  ролью **broadcaster** (то есть из настроек расширения в Twitch
  Dashboard стримера) — из обычного окна браузера сохранение работать не
  будет ("standalone mode"), это ожидаемо и совершенно нормально при
  локальном тестировании вёрстки.
